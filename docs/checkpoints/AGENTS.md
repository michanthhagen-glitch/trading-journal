# Checkpoints DOX

## Purpose

Owns dated progress snapshots for handoff and restore context.

## Ownership

- `YYYY-MM-DD-*.md`: one checkpoint or save-progress note.

## Local Contracts

- Checkpoints are snapshots, not living specs.
- Include what exists, key decisions, checks run, and useful next steps.
- Mark stale information plainly if editing an older checkpoint.

## Work Guidance

- Keep wording practical and easy to scan.
- Do not store secrets, local private data, or large logs here.

## Verification

- No separate verification for checkpoint-only edits.

## Child DOX Index

No child DOX files.
