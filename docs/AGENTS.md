# Docs DOX

## Purpose

Owns project notes, checkpoints, and handoff-style documentation.

## Ownership

- `checkpoints/`: dated project progress snapshots.

## Local Contracts

- Keep docs short, factual, and current.
- Checkpoints can become historical; do not rewrite them as current truth unless explicitly updating that checkpoint.
- Prefer a new dated checkpoint over editing old progress notes.

## Work Guidance

- Use exact commands, paths, and dates.
- Put code ownership rules in AGENTS.md files, not checkpoint notes.

## Verification

- Run `npm run check` only when doc edits are part of code changes.

## Child DOX Index

- `checkpoints/AGENTS.md` - dated progress snapshots.
